# Impeccable Context

## Design Context

### Users

Bookmark Queue Agent serves technical and privacy-conscious knowledge workers who capture many links while researching, coding, writing, and organizing personal knowledge. They use the extension inside Chrome while deciding whether a captured bookmark should be classified, moved, ignored, archived, retried, or exported.

### Brand Personality

Calm, precise, trustworthy. The UI should reduce anxiety around automation, credential setup, remote provider calls, and bookmark mutation. It should feel like a compact command center rather than a flashy AI product.

### Aesthetic Direction

Use a warm, paper-like light interface with deep ink text, restrained indigo primary actions, and semantic status colors. The experience should prioritize scanability, safe defaults, reversible actions, and clear next steps. Avoid generic SaaS dashboards, neon AI gradients, glassmorphism, and purely decorative cards.

### Design Principles

1. Make the next safe action obvious.
2. Surface trust boundaries before users need to worry about them.
3. Reveal advanced power progressively.
4. Design for long URLs, long folder paths, errors, and empty queues.
5. Keep dense extension UI readable through rhythm, hierarchy, and consistent tokens.
